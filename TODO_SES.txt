# TODO: Configuración de AWS SES

## Acción requerida antes de usar notificaciones por email

Para que el sistema de notificaciones funcione en AWS, debes verificar el email de origen en SES.

### Pasos:

1. Configurar AWS CLI con tus credenciales (si aún no está configurado):
   ```bash
   aws configure
   ```

2. Verificar el email de origen en SES:
   ```bash
   aws ses verify-email-identity --email-address noreply@boyhappy.cl
   ```

3. Revisar tu bandeja de entrada de noreply@boyhappy.cl y hacer clic en el link de verificación que envía AWS.

4. (Opcional) Si estás en Sandbox mode de SES, también debes verificar los emails de destino que quieras usar para pruebas:
   ```bash
   aws ses verify-email-identity --email-address tu-email@ejemplo.com
   ```

5. Para producción, solicitar salir de SES Sandbox mode en la consola de AWS.

### Verificar estado de verificación:
```bash
aws ses list-verified-email-addresses
```

### Notas:
- El email configurado actualmente es: noreply@boyhappy.cl
- Este email está definido en lib/boy_happy-stack.ts línea 143
- Si deseas cambiarlo, modifica SOURCE_EMAIL en el stack y redeploy
